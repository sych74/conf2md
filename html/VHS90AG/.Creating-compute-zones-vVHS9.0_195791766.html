Created by Roksoliana Ben, last modified on May 02, 2024

Compute zones can be used to create different tiers of service and have data stores and networks attached to them. The combination of compute resource, data store, and network groups can be used to create private clouds for customers. Compute zones have types which are inherited by the compute resources in the zone.

Ignore the options specified in your Control Panel but not mentioned in the documents.

Create Compute Zone

After you create a compute zone you need to attach compute resources, networks and backup servers to it.

Keep in mind that you can attach only those resources that have the same type as the compute zone. 

To create a new compute zone:

  1. Go to your Control Panel > Admin > Settings menu.
  2. Click the Compute Zones icon.
  3. Click or the Create Compute Zone button in the lower-right corner.
  4. On the screen that follows the parameters you need to input depend on the type of the compute zone you want to create:
    • Create Compute Zone
    • Create Virtual Compute Zone
  5. After you fill in all the parameters, click the Save button.

Create Virtual Compute Zone

Use only the Extended CPU Configuration option for the CPU model configuration in the Beta version.

  • Label - give your compute zone a name.

  • Server type - choose the server type from the drop-down box. Choose the virtual server type to create a KVM zone.

    The zone's type cannot be changed after the zone is created.

  • Release resource type - this option allows you to free up the compute resource resources and over-commit RAM, CPU, and CPU shares by means of the virtual machines that are shut down. By default, the compute zone is created with the Memory Guarantee option enabled. In this case, the over-committing cannot be used. 

    • Memory guarantee - the actual free compute resource memory is calculated. All virtual machines residing on the compute resource will be able to start. 

      Select only the Memory guarantee option for Release resource type in the Beta version.

  • Max VS to start at once - specify the maximum number of virtual machines that can be started simultaneously on a compute resource (5 recommended). This option ensures that virtual machines with VIP status will be booted prior to other machines.

  • Placement type - specify the compute resource selection algorithm, which will be used on virtual machine provisioning and recovery, per compute zone.

    Select only the Take HV with maximum free RAM (Sparse) option for Placement type.


    • Take HV with maximum free RAM (Sparse) - set this type to select the compute resource with maximum free RAM during the VM recovery. This option allows you to perform faster migration of virtual machines with a lesser (sparse) number of iterations during the failover.
      This option behaves in different ways, depending on the event:
      On provisioning, the round-robin algorithm will be used on compute resource selection.
      On recovery, the compute resource with maximum free RAM will be selected.

  • Failover timeout - the time period for which the iterations will run during the failover if the compute resource does not respond.

  • CPU units - set the number of CPU units that will be assigned to each compute resource in this zone by default. 

  • CPU guarantee - move the slider to the right to ensure there is enough CPU on the compute zone to create a new VM. 

  • Run sysprep - move the slider to enable Windows virtual machine deployment with running sysprep. If there are several simple deployed virtual machines from the same template in the cloud, having identical SIDS, joined to the Active Directory Domain, it will provoke the system conflict.

  • Custom Config - specify any custom commands you want to run when a compute zone is booted. 

Hot Migration Options

  • Auto-converge - select the preferable type of the auto-converge functionality:
    • Minor - auto-converge with default CPU throttling during migration.
    • Moderate - auto-converge with moderate CPU throttling.
    • Aggressive - auto-converge with aggressive CPU throttling.
