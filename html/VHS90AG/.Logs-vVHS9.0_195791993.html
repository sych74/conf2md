Virtuozzo Hybrid Server 9.0 Admin Guide : .Logs vVHS9.0

Created by Roksoliana Ben, last modified on May 02, 2024

OnApp logs all cloud management actions that take place on cloud resources, including virtual machines, disks, data stores, compute resources, templates, networks. 

Ignore the options specified in your Control Panel but not mentioned in the documents.

View and Manage Logs

To access and manage logs, go to your Control Panel > Admin > Logs menu, where you can view the log of all transactions in the cloud. The Activity Log table provides the following details:

  • Icon - indicates the status of an action.

  • Ref - the number that you can click to view details of a specific transaction.

  • Date - time when the action was performed.
  • Action - the name of the transaction.

  • Target - cloud resource with an identification number that you can click to view its details.

    You can view details on target cloud resources that are displayed in bold, indicating that they are available in the cloud. You cannot view details on resources that are unavailable, for example, resources that failed to be created.

  • Dependent - the reference number of the previous transaction on which the current transaction depends.


On the Logs page, you can click:

  • Hide Successful Market Notifications to remove from the list actions that were performed successfully. Click this button again to show the full list. 
  • Complete, Running, Failed, or Cancelled to filter logs by their status.
  • Failover Processes to view the list of failover logs. See the Failover Processes section for details.
  • Clean Logs to clear the logs completely.
  • Cancel All Pending Tasks to cancel all tasks scheduled for completion.
  • Cancel All Pending Backups to remove all pending backups.

To search for a specific transaction, you can use the search box at the top.

You can cancel a TakeBackup transaction, which is running longer and exceeding the time set for a zombie transaction and for virtual machines.

You can also relegate "pending" transactions to the failed status. To do so, hover over the pending status icon of a transaction, click that appears. This option becomes available if the transaction has been pending for the period of time specified in the Settings > Admin > Configuration > Zombie transaction time parameter.

You can see transaction logs updated in real time. This is achieved by means of the tail -f Unix command, which causes the tail command to not stop when the end of the file is reached but rather to wait for additional data to be appended to the output.

To enhance readability, the following log items are pointed out with color and font size:

  • Remote Server
  • Fatal
  • Executing Rollback

Change Tracking

To view changes that have been made during a transaction, ensure you have the appropriate Resource Diff permissions enabled.

If the size of certain resources is changed, the logs will contain information about both the current and previous size of the resource. Change tracking is available for the following transactions:

  • Resize Disk - changes of the disk size.
  • ResizeVirtualServer - changes to a VM's number of CPU cores, priority value, and RAM size.
  • ResizeApplicationServer - changes to an application server's number of CPU cores, priority value, and RAM size.
  • ResizeVirtualServerwithoutreboot - changes to a VM's number of CPU cores, priority value, and RAM size performed without a reboot.
  • ResizeApplicationServerwithoutreboot - changes to an application server's number of CPU cores, priority value, and RAM size performed without a reboot.
  • UpdateResourcePool - changes to the resource pool's resources.
  • EditFirewallRule - changes to the firewalls.
  • EditNATRule - changes to the NAT rules.
  • EditIPSECVPNRule - changes to the IPSECVPN rules.
  • HotMigrate - changes to the ID, label, and IP address of the source/destination compute resource when the VM is online.
  • ColdMigrate - changes to the ID, label, and IP address of the source/destination compute resource when the VM is offline.

To view resource changes in these transactions:

  • For all resources in the cloud: go to the Activity Log section of the Control Panel page or your Control Panel > Admin > Logs > Ref number.
  • For a virtual machine's resources: go to your Control Panel > Cloud > Virtual Servers > Label > Activity Log > Ref number.

At the beginning of the page that loads, you will see a table with the list of resources with their values before and after the transaction.

Failover Processes

Failover processes show the list of failover logs that take place on the compute zones. 

To view the list of failover processes:

  1. Go to your Control Panel > Admin > Logs menu.
  2. Click the Failover Processes button. On the page that appears, you can see the following information for each failover log:
    • Failover number
    • Indication of the time when it started 
    • Compute zone on which the failover happened
    • Time of the last iteration
    • Failover action status: active or completed


To view the failover transaction details, click its reference number. For more information on failover, refer to the Failover configuration section of this guide.
