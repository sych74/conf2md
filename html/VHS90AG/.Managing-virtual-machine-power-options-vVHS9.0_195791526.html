Virtuozzo Hybrid Server 9.0 Admin Guide : .Managing virtual machine power options vVHS9.0

Created by Roksoliana Ben, last modified on May 02, 2024

Virtual machine power options include the list of actions that you can run to change a VM power status. You can manage power options for a specific virtual machine or power on/off multiple VMs that reside on the same compute resource. 

Ignore the options specified in your Control Panel but not mentioned in the documents.

To manage power options for a specific virtual machine:

  1. Go to your Control Panel > Cloud > Virtual Servers menu.
  2. Click the label of the required virtual machine. 
  3. Click the Actions button on the VM's screen and point to Power to see the following power actions on VM's (the exact list shown depends on the VM status): 

Rebooting Virtual Machine

  1. Click the Reboot Virtual Server button.
  2. Click the OK button in the pop-up box to confirm the reboot. 

Rebooting in Recovery

  1. Click the Reboot in Recovery button.
  2. Click the Yes button in the pop-up box to confirm the reboot in the recovery mode. 

For VMs with password encryption enabled, the temporary login is "root" and the password is "recovery".

For VMs with password encryption disabled, the "root" password will be used to reboot the VM in recovery.

Windows virtual machines boot from the Linux-based recovery template in the recovery mode. You need to log in as admin via SSH or VNC console and mount a Windows system disk manually.

You cannot work with the "whole" disk (like mount -t ntfs-3g /dev/sdb1) while mounting and checking block devices inside the recovery image as the Windows disk is split into partitions.

Suspending/Unsuspending Virtual Machine

  1. Click the Suspend button to stop a VM, change its status to suspended, and disable all the major actions on VM, unless unsuspended.

    The virtual machine is suspended immediately after clicking the Suspend button without an additional confirmation.

  2. Click the Unsuspend button to activate the suspended VM and enable all the major actions. 

For more information on actions available on a suspended VM, refer to the Managing Suspended Virtual Machine section.

Shutting Down Virtual Machine 

You can shut down a specific virtual machine or multiple VMs that reside on the same compute resource. For information on shutting down multiple VMs, refer to the Power On/Off Multiple Virtual Machines section below. 

To shut down a particular virtual machine:

  1. Click the Shut Down Virtual Server button.

  2. In the pop-up box, select either the Gracefully shutdown option to terminate the VM gracefully or the Power OFF option to terminate the VM forcefully.

    If you select the Gracefully shutdown option and the system fails to shut down the VM gracefully in the time period indicated in the Timeout Before Shutting Down VMs (30-600 sec) parameter at Admin > Settings > Configuration > System, the VM will be shut down forcefully.

  3. Click the Apply button to shut down the VM. 

Starting up Virtual Server

You can start up a specific virtual machine or multiple VMs that reside on the same compute resource. For information on starting multiple VMs, refer to the Power On/Off Multiple Virtual Machines section below. 

To queue a start-up action for a VM that is currently powered off, click the Startup Virtual Server button.

When you start up a VM, it might be implicitly-migrated if the current compute resource does not have sufficient resources.

Starting up on Recovery

To start a VM in the recovery mode:

  1. Click the Startup on Recovery button to start up a VM in the recovery mode. 
  2. In the pop-up box, click the Yes button to confirm the startup. 

For VMs with password encryption enabled, the temporary login is "root" and the password is "recovery".

For VMs with password encryption disabled, the "root" password will be used to start up the VM.

Booting from ISO

You can boot virtual machines from your own ISOs or the ISOs that are uploaded and made publicly available by other users. If you boot a VM from an ISO with a RAM requirement larger than the VM's RAM, the transaction will fail. Make sure that you have enabled the Any power action on own virtual servers and Allow own virtual servers to boot from ISO permissions for the user to have access to this feature.

To boot a VM from an ISO: 

  1. Click the Boot from ISO button. 
  2. Select the ISO image from which the VM will be booted. 
  3. Click the Boot button. 

As soon as you boot a VM from the installation ISO, OnApp may lose control of any components (backups, networks, disks). The migration option is not available for VMs booted from ISO. The only available actions will be to start and stop a VM. Be aware that all the contents of the disk may be also deleted.

Powering On/Off Multiple Virtual Machines

To power on and power off virtual machines that are run on the same compute resource, follow the next steps: 

  1. In the Settings section, expand the Compute Resources menu and click a label of a compute zone where a target compute resource runs.
  2. Click a label of a compute resource. On the screen that appears, you will see a list of all virtual machines hosted on the compute resource.

  3. Choose virtual machines that you want to power on or power off by selecting the required checkboxes in the first column of the table. 

    • To select all virtual machines residing on the compute resource, click the first checkbox. To cancel the selection of all virtual machines, click this checkbox again.

    • If you select all virtual machines, only the powered-off VMs will be powered on, while the already powered-on VMs will be skipped and vice versa.

Depending on the current power status of the selected VMs, one of the following options will become available. 

Power On

To power on the selected VMs: 

    • Click the Power On button.
    • In the pop-up box, click the YES button to confirm your action.
    • As a result, multiple transactions will be scheduled to start up all the selected virtual machines one by one.


Power Off 

To power off the selected VMs: 

    • Click the Power Off button.
    • In the pop-up box, select one of the following methods: 
      • Gracefully shutdown - to run a graceful shutdown of VMs.
      • Power Off - to run a forceful shutdown of VMs.
    • Click the Submit button to confirm your action.
    • As a result, the multiple transactions will be scheduled to power off all the selected virtual machines one by one.

The bulk power on/off actions are available only to virtual machines that are run on KVM compute resources.
