Virtuozzo Hybrid Server 9.0 Admin Guide : .Quantity of disks per node vVHS9.0

Created by Roksoliana Ben, last modified on May 07, 2024

Each management node must have at least two disks (one for system and metadata, one for storage). Each secondary node must have at least two disks (one for the system, and one for storage). We recommend having at least three but not more than five metadata disks in a cluster.

The more disks per node, the lower the CAPEX. For example, a cluster created from ten nodes with two disks in each will be less expensive than a cluster created from twenty nodes with one disk.

Generally, a cluster with many nodes and few disks per node offers higher performance, while a cluster with the minimum number of nodes (3) and many disks per node is cheaper. Refer to the following table for more details.

Cluster composition recommendations
