One of the strongest features of the Virtuozzo Storage cluster is scalability. The bigger the cluster, the better Virtuozzo Storage performs.

To achieve the best performance, keep at least 20 percent of the cluster capacity free.

Three servers (minimum for data redundancy)

Three servers are required to test all of the product features.

The minimum configuration ensures that the cluster can survive the failure of one node without data loss.

The minimum cluster configuration must have at least three metadata services running. SSD disks can be assigned the system, metadata, and cache roles at the same time, freeing up more disks for the storage role.

Five or more servers (recommended for high availability and optimal TCO)

For a production environment, at least five nodes are required to ensure that the cluster can survive the failure of two nodes without data loss. We recommend creating production clusters from at least ten nodes for improved resilience, performance, and fault tolerance.

A production-ready cluster can be created from just five nodes using the recommended hardware. However, we suggest entering production with at least ten nodes if you aim to achieve significant performance advantages over direct-attached storage (DAS) or improved recovery times.

The recommended cluster configuration must have five metadata services running and dedicated disks for write cache.
