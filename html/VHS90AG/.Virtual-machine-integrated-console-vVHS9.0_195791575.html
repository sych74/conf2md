Virtuozzo Hybrid Server 9.0 Admin Guide : .Virtual machine integrated console vVHS9.0

Created by Roksoliana Ben, last modified on May 02, 2024

OnApp provides an integrated VNC console that gives users direct access to virtual machines through the Control Panel UI. The noVNC console is provided for virtual machines built on KVM based on WebSockets. Users with the Administrator role can access all virtual machine consoles for support and troubleshooting purposes. The console connects a user browser to a VNC port or VNC WebSocket port available via a compute resource for the guest console. 

Ignore the options specified in your Control Panel but not mentioned in the documents.

There are two ways to access the virtual machine VNC console via the Control Panel:

  1. Go to your Control Panel > Cloud > Virtual Servers menu. 
  2. Click the label of the destination virtual machine.
  3. Click the Console tab.

      OR

  1. Go to your Control Panel > Cloud > Virtual Servers menu. 
  2. Click the Actions button next to the destination virtual machine and select Console.

For the HTML5 console, click the Re-connect button if the connection is lost. The re-connection to the console runs as follows: 

  • If the console runs as expected, clicking the Re-connect button causes disconnection and the console is re-connected automatically after 1.5 seconds.
  • If the console gets stuck, clicking the Re-connect button runs your request once again and re-connects the console without reloading.
  • If the console gets disconnected with a status code and an error message, the console is re-connected automatically after 1.5 seconds.

To use the Java console instead of HTML5, go to Admin > Settings > Configuration and edit settings on the System tab. For more information, refer to System Configuration.
