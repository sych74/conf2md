Virtuozzo Hybrid Server 9.0 Admin Guide : .Virtual machine transactions and logs vVHS9.0

Created by Roksoliana Ben, last modified on May 02, 2024

The system records a detailed log of all the transactions taking place on your virtual machines. The list of transactions logged by the system includes:

  • Provision virtual server
  • Startup virtual server 
  • Stop virtual server
  • Resize virtual server without reboot
  • Configure Operating System
  • Build disk
  • Resize disk
  • Format disk
  • Destroy disk
  • Take backup
  • Convert backup
  • Restore backup
  • Destroy backups
  • Destroy virtual server
  • Destroy template
  • Download template
  • Update firewall

Ignore the options specified in your Control Panel but not mentioned in the documents.

To view transactions for a virtual machine:

  1. Go to your Control Panel > Cloud > Virtual Servers menu.
  2. Click the label of the virtual machine you're interested in.
  3. The details screen for that virtual machine shows recent transactions in the Activity Log section. 

To cancel all pending tasks, click the Cancel All Pending Tasks for this virtual server button.

To cancel the TakeBackup transaction which is running longer and exceeding the time set for a zombie transaction:

  1. Go to your Control Panel > Cloud > Virtual Servers menu.
  2. Click the label of the required virtual machine.

  3. On the following page, in the Activity Log section, hover over the icon of the transaction and then click that appears on the icon of this transaction.

    Please note that this option becomes available if the transaction has been running for the period of time specified in the Zombie transaction time parameter, which can be found at your Control Panel > Admin > Settings > Configuration icon.

  4. In the dialog box, click Proceed.

    The rollback is executed starting from the command that has been canceled. 

You can also view the details of a particular log item by clicking its reference number. The following page shows the log output and the details:

  • Date - time in the [YYYY][MM][DD]T[hh][mm][ss]Z format.
  • Action - the action name.
  • Status - the action status (Complete, Warn, Pending, or Failed).
  • Ref - the log item's reference number.
  • Target - the action target.
  • Dependent - the reference number of the previous transaction on which the current transaction depends.
  • Started at - the time when the action is started.
  • Completed at - the time when the action is completed.
  • Data Store - the datastore label.
  • Virtual Server - the virtual machine label.
  • Compute resource - the label of compute resource.
  • Initiator - the user who initiated the action.

If you want to see only the detailed output, you can hide log info with the arrow button in the upper-right corner.
